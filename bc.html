Method

I run the panning (cf our paper and in the algorithm implementation ) on the dataset Micro-RNA expression signatures in breast cancer progression (see here ). This dataset has \(n=156\) patients that we split into training and test sets, \(n_{\text{Training}} = 130\) and \(n_{\text{Test}}=26\). There are \(p=15739\) RNA expressions per patients, we eliminate those with missing values so we worked with \(p=14251\). We have experienced mitigated results with the original version of the panning. I tried two new version of the panning:
1. From the first step I screen only the variables below a given quantile \(\hat{q}\) on the cross-validation errors (here I used \(\hat{q}=0.00835\)). I obtain a much small number of variables (here 118 remaining ones) that I save into a set, say \(\mathcal{S}_1\). From these variables in \(\mathcal{S}_1\), I construct exhaustively all possible models of size 2 (here 6903) and compute their CV errors. I save models with errors below the quantile \(\hat{q}\) into \(\mathcal{S}_2\). Then, for the rest of the algorithm, I iterate for \(2< d<d_{\text{max}}\): I take the models in \(\mathcal{S}_{d-1}\) and construct exhaustively all possible combination of models size \(d\) by adding any (non-redundant) variables from \(\mathcal{S}_1\). Hence only the first variables selected in \(\mathcal{S}_1\) can potentially appears, all the other ones are discared.
2. From the first step I attribute to every unit (here variable) an individual probability of being selected \(\pi_i,i=1,\dots,p\). This probability is defined as \(\hat{\pi}_i = (\max{\text{CV}} - 1) / \max{\text{CV}}\), where \(\max{\text{CV}}\) is the maximum CV error observed. The variables are stored in a set \(\mathcal{S}\) with associated probabilities set \(\mathcal{P}\). Then, for the rest of the algorithm, I iterate for \(2\leq d<d_{\text{max}}\): I construct a model of size \(d\) by drawing units from \(\mathcal{S}\) with probability in \(\mathcal{P}\), and evaluate its CV error, repeat \(B\) times (here \(B=10^4\)); the set \(\mathcal{S}\) is augmented with \(B\) new units (now models of size \(d\)) and the probabilities are all reevaluated in \(\mathcal{P}\).

Remark:
* The first step is an exhaustive computation of all cross-validation for each individual variables.
* For both versions of the algorithm I stop at \(d_{\text{max}} = 15\).

Results of the algorithm

I represent boxplot of the CV errors of the selected models and the related testing error (unknown a prior). Here are the results with the first version of the panning:

load("~/Documents/Projects/Panning/bc/Testing.rda")
cvs1 <- vector("list",15)
for(i in 1:15){
  cvs1[[i]] <- Testing[[i]][,1]
}
tes1 <- vector("list",15)
for(i in 1:15){
  tes1[[i]] <- 1-Testing[[i]][,6]
}

par(mfrow=c(1,2))
boxplot(cvs1,col="gray75",main="CV error",ylim=c(0,.6))
boxplot(tes1,col="gray75",main="Testing error",ylim=c(0,.6))

We can remark:
* We observe the Nike sign.
* CV errors seem not too accurate as an estimate of the testing error.

And now the second version:

load("~/Documents/Projects/Panning/bc/Testing2.rda")
dmax <- 15
cvs2 <- vector("list",dmax)
for(i in 1:dmax){
  cvs2[[i]] <- Testing[[i]][,1]
}
tes2 <- vector("list",dmax)
for(i in 1:dmax){
  tes2[[i]] <- 1-Testing[[i]][,6]
}

par(mfrow=c(1,2))
boxplot(cvs2,col="gray75",main="CV error",ylim=c(0,.6))
boxplot(tes2,col="gray75",main="Testing error",ylim=c(0,.6))

We can remark:
* Much more variability in CV and testing errors compare to version 1 (more variables).
* CV errors decreasing slower than version 1.

A compromise between these two versions might be a good option for future research.

Main results

I keep arbitrarly models with CV errors below 6% (the observed minimum being approx. 5.36%). This level excludes result from the version 2 of the panning (observed minimum at 8.78%).

Graphically, it means we are taking the following shaded area:

id_x <- which(sapply(cvs1,function(x)sum(x<.06))!=0)
y_1 <- c(sapply(cvs1,max),sapply(cvs1,min)[dmax:1])
y_2 <- rep(NA, 2 * length(id_x))
for(i in seq_along(id_x)){
